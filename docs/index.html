Built around the operations you repeat every day.

No service dashboard maze. Pick a profile, choose a region, jump to a prefix, and work from a file-browser layout.

01

Navigate by bucket, folder, or typed prefix.

Use normal browsing for discovery, then paste exact S3 prefixes when you already know where you need to be.

02

Move objects without losing context.

Upload, download, delete, and refresh from the same workspace while the inspector keeps the selected target visible.

03

Invalidate CloudFront from the same place.

Create invalidations next to the S3 content they affect, using local AWS credentials and your selected profile.

# local-first by design
aws profiles → S3 Studio
S3 buckets   → object browser
prefixes     → typed navigation
ACLs         → loaded on demand
CloudFront   → invalidation panel

No hosted sync. No account import.

Permissions stay out of the way until they matter.

ACL work is heavy and easy to get wrong. S3 Studio keeps it scoped to the selected object or prefix and treats it as an explicit access-settings task.

Folders are treated like prefixes, not fake files.

The interface focuses on the prefix path, child object count, and relevant actions instead of wasting height on previews that cannot render anything useful.

Native desktop packaging, open source internals.

Built with Tauri, Rust, React, and TypeScript. The app talks directly to AWS using the same local profile setup you already use in the CLI.

Download a build for your machine.

Each tagged release is built by GitHub Actions for desktop platforms. Linux ships as deb/rpm packages to avoid flaky AppImage bootstrap downloads.
